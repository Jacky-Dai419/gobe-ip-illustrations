# 单批多图生成合同

## 任务定义

把一篇文章的完整配图计划作为一个批次：一次上传参考图，一次发送包含全部编号任务的总提示词，随后自动保存返回的图片。

## 数量

- 单批最多 10 张，封面、正文图和总结图都计入。
- 同时需要封面和总结图时，正文图最多 8 张。
- 认知锚点过多时，只保留核心判断、必要流程、关键转折和强隐喻。

## 总提示词

总提示词包含：

1. 全部参考图共同定义同一个人物。
2. 白底、手绘、怪诞但清爽的整组风格。
3. 每张图片的编号、角色、场景、动作、比例、短标签和文件名。
4. 每个编号输出一张独立图片，不合并成九宫格、拼贴、漫画页或多面板图。
5. 模型可以自由调整姿势、道具、画面细节和隐喻表现。

## Python 浏览器执行

运行 `scripts/generate_with_chatgpt_web.py`。程序控制已登录的 ChatGPT 网页：新建临时任务、保留用户手动设置的模型与思考程度、明确选择“创建图片”、上传参考图组、一次发送完整批次、等待生成、下载图片，并在成功后删除任务。程序不得自动切换模型档位。

ChatGPT 可能把整批结果显示成“1 张主图 + 右侧多张缩略图”的图片组。程序必须进入并读取整个 Assistant 图片组，按 GPT 原图资源地址去重后下载；不能只看正文主图数量，也不能因为缩略图在页面上只显示为 48px 就丢弃其对应的全尺寸原图。

只有实际下载数量等于视觉计划数量，批次才算技术完成。数量不足不是画面评审，不触发逐张返工；必须把本次运行记录为失败并保留临时任务供恢复。只有完整计划数量下载并验证成功后才能删除任务。

程序不建立图片评审、人物一致性评分、文字识别、比例检测、二次提示或重生成环节。网页没有返回任何可保存图片时作为技术错误退出；其他已返回结果直接落盘。等待上传、生成、下载和确认删除结果属于技术状态处理，不属于画面检查。

## 手动执行

浏览器底座不可用时，在同一个 ChatGPT“生成图片”任务中上传参考图并发送一次总提示词。生成结束后直接下载图片，不插入逐张确认或返工对话，并删除临时任务。
