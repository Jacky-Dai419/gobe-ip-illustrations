# 参考图合同

## 两类参考

- `IP identity reference`：定义人物是谁。
- `style reference`：定义画面怎样画。

替换 IP 图片后继续使用同一套文章理解、构图和提示模板。

## 目录

```text
references/
├── ip/
│   ├── source/      # ChatGPT 网页直接上传的 PNG、JPG、JPEG、WEBP
│   └── originals/   # HEIC 等原件，不上传到网页
└── style/           # 风格规则与可选风格图片
```

`references/ip/` 中的全部兼容图片共同定义一个人物，可以包含正面、侧面、背面、表情和动作视图。风格图只控制线条、留白、色彩克制和画面气质。

## Python 网页传递

`scripts/generate_with_chatgpt_web.py` 递归读取参考图，按 SHA-256 跳过完全相同的副本，再让浏览器底座把整组文件挂到 ChatGPT 网页同一个上传框。完整批量提示词随后只发送一次。

参考图顺序固定为：IP 图在前，可选风格图在后。在提示词中说明所有 IP 图都是同一人物的不同参考视图，并明确命令模型严格使用该人物，保留可识别的脸部、发型、眼镜、轮廓、比例和标志性服装。

## 手动兜底

在“生成图片”模式中上传全部 IP 图片，再粘贴一次批量提示词。不要拆分成多轮追加要求。

网页自动化与手动兜底遵守同一条规则：图片必须真的上传到 ChatGPT，不能只在提示词中描述本地文件。

## 自由度

参考图固定人物身份，文章固定核心意思；姿势、道具、局部构图、细节和隐喻表达交给图像模型自由发挥。生成后直接保存结果，不建立人物一致性、画面质量或风格评分流程。
