# 构图与隐喻规则

## 结构类型

每张图只选择一种主结构：

- 工作流：输入、处理、输出。
- 系统局部：只展示 3–5 个关键部件。
- 前后对比：混乱与有序、手动与自动、卡住与跑通。
- 角色状态：用 2–4 个动作展示状态变化。
- 概念隐喻：用一个怪诞但成立的大物件表达概念。
- 方法分层：用自然堆叠、盒子或工位表示层级。
- 地图路线：用路径、门、桥或接力表示过程。
- 小漫画：用 2–4 个连续动作表达转折。

## 原创隐喻

按三步组合：

1. 把抽象概念换成物理动作：卡住、漏掉、拉回、分拣、沉淀、拆开、折叠、回流。
2. 把系统换成低科技物件：抽屉、纸箱、水管、门、秤、井、梯子、转盘、怪机器。
3. 让参考 IP 承担动作：拉、扛、塞、捞、压、称、缝、剪、守、推、接、拆或标记。

只选少量物件。怪诞感来自关系，不来自堆满奇怪物品。

## 反复刻

- 不复刻任何参考图的完整构图。
- 不因为参考 IP 做过某个动作，就在每张图重复这个动作。
- 同一组图片避免连续使用相同机器、管道、箭头或角色站位。
- 画面先服务文章认知，再服务 IP 展示。
